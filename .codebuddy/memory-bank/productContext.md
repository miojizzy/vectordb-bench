# Product Context

## Problem Statement

向量数据库在 AI 应用中扮演重要角色，但选择合适的数据库需要：
1. 性能数据支持决策
2. 不同场景下的表现对比
3. 资源配置优化指导

现有的 Python 版 VectorDBBench 存在：
- 性能开销较大
- 并发测试效率有限
- CPU 资源控制不够精细

## Solution

C++ 版本 VectorDBBench 提供：
- 低开销的测试框架
- 精确的 CPU 资源控制
- 高效的多数据库对比

## User Stories

### US-1: 单数据库性能测试

> 作为数据库开发者，我想测试我的数据库在不同 CPU 配置下的性能表现，以便优化配置。

**验收标准**：
- 支持指定 CPU 核心数
- 支持指定线程数
- 输出 QPS、延迟、召回率等指标

### US-2: 多数据库对比

> 作为算法工程师，我想对比多个向量数据库的性能，以便选择最适合的产品。

**验收标准**：
- 支持同时配置多个数据库
- 使用相同数据集测试
- 生成对比报告

### US-3: 容量测试

> 作为运维人员，我想测试数据库的最大容量，以便进行容量规划。

**验收标准**：
- 自动插入数据直到内存满
- 记录最大加载数量
- 测试不同维度向量的容量

### US-4: 过滤搜索测试

> 作为开发者，我想测试带过滤条件的搜索性能，以便了解实际业务场景下的表现。

**验收标准**：
- 支持 int 类型过滤
- 支持 label 标签过滤
- 记录过滤条件对性能的影响

## Non-Functional Requirements

### NFR-1: Performance

- 测试开销不超过被测系统 5%
- 支持并发 100+ 查询
- 数据加载吞吐量 > 100K vectors/sec

### NFR-2: Reliability

- 测试失败自动重试
- 异常结果检测
- 资源泄漏检测

### NFR-3: Usability

- 命令行参数清晰
- 配置文件格式直观
- 错误信息友好

### NFR-4: Maintainability

- 模块化设计
- 接口抽象
- 单元测试覆盖 > 80%
